### Task 1: Fix `sys.path` shadowing in compatibility tests

The compatibility conftest adds `vendor/mlflow` (project root) to `sys.path` at line 16. `test_registry_compat.py` does the same at line 20. This causes `import mlflow` to resolve to the vendored source tree (which needs `opentelemetry` etc.) instead of the installed `mlflow` package. Only `vendor/mlflow/tests` should be on `sys.path` — the `tests.__path__` extension at line 28 already handles `from tests.*` imports.

**Files:**
- Modify: `tests/compatibility/conftest.py:13-16`
- Modify: `tests/compatibility/test_registry_compat.py:20-21`

- [ ] **Step 1: Run existing compat tests to confirm the breakage**

Run: `uv run pytest tests/compatibility/test_registry_contract.py --co -q 2>&1 | tail -5`
Expected: `ModuleNotFoundError: No module named 'opentelemetry'` (vendored `mlflow` shadows installed package)

- [ ] **Step 2: Remove the `vendor/mlflow` sys.path insertion from conftest**

In `tests/compatibility/conftest.py`, remove the `_VENDOR_MLFLOW` variable and its `sys.path.insert` call. Keep `_VENDOR_MLFLOW_TESTS` — it's used by `_VENDOR_TESTS_PATH` on line 38.

Replace lines 13-16:
```python
_VENDOR_MLFLOW = str(Path(__file__).parents[2] / "vendor" / "mlflow")
_VENDOR_MLFLOW_TESTS = str(Path(__file__).parents[2] / "vendor" / "mlflow" / "tests")
if _VENDOR_MLFLOW not in sys.path:
    sys.path.insert(0, _VENDOR_MLFLOW)
```

With:
```python
_VENDOR_MLFLOW_TESTS = str(Path(__file__).parents[2] / "vendor" / "mlflow" / "tests")
```

- [ ] **Step 3: Remove the `vendor/mlflow` sys.path insertion from test_registry_compat.py**

In `tests/compatibility/test_registry_compat.py`, remove lines 20-21:
```python
sys.path.insert(0, str(Path(__file__).parents[2] / "vendor" / "mlflow"))
```

The conftest's `tests.__path__` extension already handles `from tests.store.*` imports.

### Task 2: Add `workspaces_enabled` fixture to compatibility conftest

The vendored gateway tests depend on a `workspaces_enabled` autouse fixture. Since the DynamoDB store is always workspace-aware, add this to the shared conftest so it works for all compat tests.

**Files:**
- Modify: `tests/compatibility/conftest.py` (add imports and fixture at end of file)

- [ ] **Step 1: Add the `workspaces_enabled` fixture**

Add `uuid` to the imports at the top of the file. Add the following at the end of `tests/compatibility/conftest.py`:

```python
from mlflow.utils.workspace_context import WorkspaceContext  # noqa: E402


@pytest.fixture(autouse=True, params=[False, True], ids=["workspace-disabled", "workspace-enabled"])
def workspaces_enabled(request, monkeypatch):
    enabled = request.param
    monkeypatch.setenv(MLFLOW_ENABLE_WORKSPACES.name, "true" if enabled else "false")
    if enabled:
        workspace_name = f"compat-test-{uuid.uuid4().hex}"
        with WorkspaceContext(workspace_name):
            yield enabled
    else:
        yield enabled
```

Note: `MLFLOW_ENABLE_WORKSPACES` is already imported on line 82. `uuid` needs to be added to the top-level imports.

Expected: Test count roughly doubles (each test now runs with both `workspace-disabled` and `workspace-enabled`). All tests pass or xfail as before. No new failures. If any existing test breaks under `workspace-enabled`, fix the issue before proceeding — do not move on with regressions.
