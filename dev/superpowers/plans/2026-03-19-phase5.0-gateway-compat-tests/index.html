    Phase 5.0: Port Gateway Compatibility Tests¶

    For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.

    Goal: Port MLflow's 55 gateway store tests into tests/compatibility/test_gateway_compat.py so all subsequent Phase 5 implementation is TDD.

    Architecture: Import test functions from vendor/mlflow/tests/store/tracking/test_gateway_sql_store.py, provide DynamoDB-backed store fixture, add workspaces_enabled fixture to shared conftest, mark all 55 tests xfail(raises=NotImplementedError). Fix existing sys.path bug that shadows installed mlflow with vendored source.

    Tech Stack: pytest, moto (mock_aws), MLflow entities, vendored MLflow test suite (git submodule)

    Spec: docs/superpowers/specs/2026-03-17-ai-gateway-design.md (Phase 5.0 section)

    Note: The spec's test inventory table says 9 model definition tests and 9 endpoint tests, but the vendored file has 10 in each category (it missed test_update_gateway_model_definition and test_update_gateway_endpoint). These are legitimate store API contract tests, so we import all 10 in each. Total: 55 tests imported, 11 excluded. The spec table should be updated after this plan lands.

    Task 1: Fix sys.path shadowing in compatibility tests¶

    The compatibility conftest adds vendor/mlflow (project root) to sys.path at line 16. test_registry_compat.py does the same at line 20. This causes import mlflow to resolve to the vendored source tree (which needs opentelemetry etc.) instead of the installed mlflow package. Only vendor/mlflow/tests should be on sys.path — the tests.__path__ extension at line 28 already handles from tests.* imports.

    Files: - Modify: tests/compatibility/conftest.py:13-16 - Modify: tests/compatibility/test_registry_compat.py:20-21

    • Step 1: Run existing compat tests to confirm the breakage

    Run: uv run pytest tests/compatibility/test_registry_contract.py --co -q 2>&1 | tail -5 Expected: ModuleNotFoundError: No module named 'opentelemetry' (vendored mlflow shadows installed package)

    • Step 2: Remove the vendor/mlflow sys.path insertion from conftest

    In tests/compatibility/conftest.py, remove the _VENDOR_MLFLOW variable and its sys.path.insert call. Keep _VENDOR_MLFLOW_TESTS — it's used by _VENDOR_TESTS_PATH on line 38.

    Replace lines 13-16: 

    _VENDOR_MLFLOW = str(Path(__file__).parents[2] / "vendor" / "mlflow")
_VENDOR_MLFLOW_TESTS = str(Path(__file__).parents[2] / "vendor" / "mlflow" / "tests")
if _VENDOR_MLFLOW not in sys.path:
    sys.path.insert(0, _VENDOR_MLFLOW)

    With: 

    _VENDOR_MLFLOW_TESTS = str(Path(__file__).parents[2] / "vendor" / "mlflow" / "tests")

    • Step 3: Remove the vendor/mlflow sys.path insertion from test_registry_compat.py

    In tests/compatibility/test_registry_compat.py, remove lines 20-21: 

    sys.path.insert(0, str(Path(__file__).parents[2] / "vendor" / "mlflow"))

    The conftest's tests.__path__ extension already handles from tests.store.* imports.

    • Step 4: Run existing compat tests to confirm they still collect

    Run: uv run pytest tests/compatibility/test_registry_contract.py --co -q Expected: Tests collect successfully (no opentelemetry error).

    Run: uv run pytest tests/compatibility/test_registry_compat.py --co -q Expected: Tests collect successfully. If from tests.store.model_registry... fails, the tests.__path__ extension needs to also cover tests.store — add "tests.store" and "tests.store.model_registry" to the _ensure_vendor_subpkg loop.

    • Step 5: Run the full compatibility suite to verify no regressions

    Run: uv run pytest tests/compatibility/ -x -q Expected: All existing tests pass (or xfail as before). No new failures.

    • Step 6: Commit
    git add tests/compatibility/conftest.py tests/compatibility/test_registry_compat.py

    Task 2: Add workspaces_enabled fixture to compatibility conftest¶

    The vendored gateway tests depend on a workspaces_enabled autouse fixture. Since the DynamoDB store is always workspace-aware, add this to the shared conftest so it works for all compat tests.

    Files: - Modify: tests/compatibility/conftest.py (add imports and fixture at end of file)

    • Step 1: Add the workspaces_enabled fixture

    Add uuid to the imports at the top of the file. Add the following at the end of tests/compatibility/conftest.py:

    from mlflow.utils.workspace_context import WorkspaceContext  # noqa: E402


@pytest.fixture(autouse=True, params=[False, True], ids=["workspace-disabled", "workspace-enabled"])
def workspaces_enabled(request, monkeypatch):
    enabled = request.param
    monkeypatch.setenv(MLFLOW_ENABLE_WORKSPACES.name, "true" if enabled else "false")
    if enabled:
        workspace_name = f"compat-test-{uuid.uuid4().hex}"
        with WorkspaceContext(workspace_name):
            yield enabled
    else:
        yield enabled

    Note: MLFLOW_ENABLE_WORKSPACES is already imported on line 82. uuid needs to be added to the top-level imports.

    • Step 2: Run existing compat tests to verify no regressions

    Run: uv run pytest tests/compatibility/ -x -q Expected: Test count roughly doubles (each test now runs with both workspace-disabled and workspace-enabled). All tests pass or xfail as before. No new failures. If any existing test breaks under workspace-enabled, fix the issue before proceeding — do not move on with regressions.

    • Step 3: Commit
    git add tests/compatibility/conftest.py
git commit -m "test: add workspaces_enabled fixture to compatibility conftest"
